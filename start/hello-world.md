# Hello World

第一次接触 `C++11` 时，`Hello World` 的价值不只是打印一句话，而是帮你确认三件事：

- 代码能被正确编译
- 你知道最小程序长什么样
- 你开始理解“写代码 -> 编译 -> 运行”这条最短路径

先看最小示例：

```cpp
#include <iostream>

int main() {
    std::cout << "Hello, C++11" << std::endl;
    return 0;
}
```

这段代码里已经包含了几个最基本的元素：

- `#include <iostream>` 表示使用标准输出能力。
- `main()` 是程序入口。
- `std::cout` 用于输出文本。
- `return 0;` 表示程序正常结束。

# 第一次编译

如果你使用 `g++` 或 `clang++`，最常见的编译命令可以先写成：

```bash
g++ -std=c++11 -Wall -Wextra -pedantic hello.cpp -o hello
```

然后运行：

```bash
./hello
```

你应该看到：

```text
Hello, C++11
```

这里最值得记住的是 `-std=c++11`。学习阶段请尽量显式写出标准版本，不要依赖默认配置。

# `Hello World` 在 C++ 里到底说明了什么

在很多语言里，`Hello World` 只是一个象征性仪式；但在 `C++` 里，它至少说明下面这些基础环节已经打通：

1. 编译器可用。
2. 标准库头文件可用。
3. 你知道如何把源文件编译成可执行文件。
4. 你开始区分“写代码”和“生成程序”是两步。

这个差别很重要，因为后面你会很快接触到：

- 多文件编译
- 链接错误
- 编译参数
- 调试信息
- 构建系统

所以不要把这一步看得太轻，它是后面所有实验的起点。

# 一个更接近现代风格的最小示例

如果你想顺手体验一点 `C++11` 的味道，可以把程序改成这样：

```cpp
#include <iostream>
#include <vector>

int main() {
    std::vector<int> values{1, 2, 3};

    for (const auto& value : values) {
        std::cout << value << std::endl;
    }

    return 0;
}
```

这段代码比纯字符串输出多体现了几件事：

- 使用标准库容器
- 使用花括号初始化
- 使用范围 `for`
- 使用 `auto`

它已经更接近真实 `C++11` 代码的常见写法。

# 第一次看懂编译错误

建议你主动做几次小实验，而不是只满足于“能跑”：

1. 删掉 `#include <iostream>`，看编译器如何报错。
2. 把 `std::cout` 改成 `cout`，观察名字未声明错误。
3. 把 `main` 拼错，看看入口函数问题是什么样。
4. 把分号漏掉一处，熟悉最基本语法错误提示。

`C++` 学习很依赖“读错误信息”的能力，这一步越早练越好。

# 第一次多文件实验

当你确认单文件程序可以编译后，建议立刻补一个最小多文件实验。

`main.cpp`：

```cpp
#include <iostream>
#include "math.h"

int main() {
    std::cout << add(2, 3) << std::endl;
    return 0;
}
```

`math.h`：

```cpp
#ifndef MATH_H
#define MATH_H

int add(int a, int b);

#endif
```

`math.cpp`：

```cpp
#include "math.h"

int add(int a, int b) {
    return a + b;
}
```

编译命令：

```bash
g++ -std=c++11 -Wall -Wextra -pedantic main.cpp math.cpp -o demo
```

这个实验能帮助你更早理解头文件、声明、定义和链接的关系。

# 第一次调试意识

初学者最容易忽略的一点是：`C++` 不只是“编译成功就结束”，你还应尽早养成调试意识。

学习阶段建议默认加上：

```bash
g++ -std=c++11 -Wall -Wextra -pedantic -g hello.cpp -o hello
```

其中 `-g` 用于生成调试信息。即使你暂时还不系统学习调试器，也建议把它作为默认习惯。

# 与其它语言的感觉差异

如果你来自 `Go`、`PHP`、`JS`、`TS`，第一次写 `Hello World` 时很可能会觉得：

- 步骤更多
- 命令更长
- 头文件和命名空间有点烦

这种感觉是正常的。因为 `C++` 从一开始就让你接触编译和标准库边界，而不是把这些细节全部藏起来。

你可以把这理解成：

> `C++` 上手摩擦更大，但你会更早接触真实工程里的关键机制。

# 建议你立刻做的 5 件小事

为了让这一章真正有用，建议你不要只看例子，而是亲手做下面 5 件事：

1. 自己新建一个 `hello.cpp` 并编译一次。
2. 把输出文本改掉，确认自己不是只在复制粘贴。
3. 故意制造一个语法错误，看看编译器提示。
4. 再做一次最小多文件实验。
5. 编译时加上 `-g` 和警告参数。

这样你会很快从“看懂示例”进入“开始有手感”。

# 小结

`Hello World` 在 `C++11` 里真正的意义，是让你打通最短实验闭环：写代码、编译、运行、看错误、再修改。只要这一条链路稳了，后面学习基础语法、函数、指针、类和构建模型时，理解速度会明显更快。
