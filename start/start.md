# 前言

如果你平时主要写 `Go`、`PHP`、`TypeScript`，第一次认真进入 `C++11`，很容易出现一种强烈的不适感：

- 看得懂很多语法，但写起来没有把握。
- 明明只是一个小功能，却突然要理解头文件、编译、链接和构建系统。
- 变量、指针、引用、对象、析构这些概念似乎都见过，但很难形成稳定直觉。
- 你能感觉到现代 `C++` 很强调“资源”和“生命周期”，但一开始不知道这和日常开发有什么关系。

这套文档就是为这种场景准备的。

它不假设你有真实 `C/C++` 项目经验，也不假设你还记得系统化的 `C++98` 基础。
它假设的是另一件事：你已经是有经验的开发者，知道程序设计、模块化、接口、数据结构和调试的基本思路，只是需要把这些能力迁移到 `C++11` 的工程语境里。

# 语言难点

对许多其它语言开发者来说，`C++` 的难点往往不在“会不会写 if/for/class”，而在下面这些地方：

- `C++` 同时保留了接近底层的能力和高层抽象能力，选择空间很大。
- 一段代码除了“功能对不对”，还经常要关心对象生命周期和资源释放。
- 看似相近的几种写法，在拷贝成本、所有权、异常安全上可能差别很大。
- 旧教程、旧项目、现代写法常常同时存在，容易形成混乱印象。

所以学习 `C++11` 的关键不是记忆更多语法，而是建立一套能解释真实项目代码的心智模型。

# 适用读者

本文档主要面向下面这些读者：

- 有其它语言开发经验，但几乎没有 `C/C++` 项目经验。
- 学过基础 `C/C++`，但现在只剩下零散印象。
- 能看懂一些简单代码，却很难独立编译、修改和调试 `C++` 程序。
- 准备接手 `C++11` 项目，希望先补齐最关键的一圈知识。

如果你已经是多年 `C++` 工程师，这套文档里的很多内容会偏基础；
但如果你正处于“有开发经验，却还没真正建立 C++ 工程直觉”的阶段，它会更适合你。

# 学习目标

阅读这套文档的目标不是“记住全部标准细节”，而是尽快做到下面几件事：

- 能搭建并运行一个最小 `C++11` 程序。
- 能理解源文件、头文件、编译、链接之间的关系。
- 能读懂常见 `C++11` 业务代码中的值、引用、指针和对象生命周期。
- 能识别哪些老写法只是历史遗留，哪些是今天仍然合理的实践。
- 能把标准库容器、算法、字符串、I/O 与面向对象设计串起来。
- 能在项目里做小到中等规模修改，而不是只敢看不敢动。

## 为什么以 C++11 为主

`C++11` 是现代 `C++` 的真正起点。很多你今天在项目里经常看到的习惯，几乎都从这一版开始稳定下来：

- `auto`
- `decltype`
- 统一初始化
- 范围 `for`
- `nullptr`
- `enum class`
- `lambda`
- 右值引用与移动语义
- 智能指针
- `constexpr`
- `override` / `final`
- `=default` / `=delete`
- `chrono`
- 线程库
- `unordered_map`

如果 `C++11` 这层基础没有建立好，继续看 `C++14/17/20` 往往只会更混乱，因为你会不断遇到“新旧写法为什么不同”却说不清楚原因。

## 与其它语言的主要差异

如果你来自 `Go`、`PHP`、`TS`，最需要尽早接受的是以下差异：

- `C++` 没有统一运行时帮你托底所有资源释放。
- **对象离开作用域时会析构，这不是边缘细节，而是核心设计基础。**
- 函数参数是按值、按引用还是按指针传递，会直接影响语义和性能。
- 容器、算法和迭代器构成了大量标准写法。
- 很多错误会在编译期暴露，但也存在运行期不会立即报错的未定义行为。

> 你可以把它理解成：`C++` 给了你更强的表达能力，也要求你更明确地表达意图。

# 文档重点

这套教程会反复围绕下面这些主题展开：

- 如何理解值语义、引用语义、所有权和 `RAII`
- 为什么现代 `C++` 不鼓励散落的 `new/delete`
- 为什么 `const`、`override`、`=delete` 这些看似小的语法其实很关键
- 如何通过标准库容器和算法写出更像现代 `C++` 的代码
- 接手历史项目时，如何看懂 `C` 风格和 `C++98` 风格遗留代码

换句话说，本文档关注的是“如何进入项目开发”，而不是“如何把所有语言点平铺背熟”。

## 文档写法风格

本文档会尽量统一采用下面这些写法：

- 中文讲解为主，保留必要的英文关键字、类型名和库名。
- 代码示例尽量短小、完整、可单独编译。
- 优先解释“为什么要这么写”，而不是只告诉你“语法是什么”。
- 同时给出部分旧写法，帮助你识别历史代码。
- 尽量站在其它语言开发者的视角解释容易误解的地方。

# 先建立 6 个直觉

如果你现在时间有限，只先记住下面 6 条，会非常有帮助：

1. 不要把裸指针默认理解为“拥有这个对象并负责释放”。
2. 离开作用域会触发析构，很多资源清理都是这样完成的。
3. 传值、传引用、传指针不是语法细节，而是接口设计选择。
4. 标准库容器和算法不是附属品，而是日常主力。
5. 旧代码能编译，不代表它是今天推荐的写法。
6. 先把 `C++11` 学扎实，再谈更高版本特性。

# 推荐阅读顺序

虽然侧边栏是按主题组织的，但如果你的目标是“尽快形成能读项目代码的基本能力”，更推荐按这个顺序：

1. 先看 `简介`、`环境准备`、`头文件与构建模型`。
2. 再看 `基础语法`、`基础类型`、`变量`、`常量`、`函数`。
3. 然后重点看 `字符串`、`数组`、`指针`、`引用`、`I/O`、`数据结构`。
4. 接着看 `异常`、`模板`、`并发`、`安全边界` 这些工程主题。
5. 最后进入 `类/对象`、`继承`、`多态`、`重载`、`接口`。

如果你已经拿到一个真实项目，也可以采用另一种方式：每学完一章，就去项目里找 2 到 3 段对应代码做映射。这样比“纯看教程”更容易形成手感。

# 一个典型的断层示例

很多其它语言开发者第一次看现代 `C++` 时，会遇到类似下面的代码：

```cpp
#include <iostream>
#include <memory>
#include <vector>

class Printer {
public:
    virtual ~Printer() = default;
    virtual void print(int value) const = 0;
};

class ConsolePrinter : public Printer {
public:
    void print(int value) const override {
        std::cout << value << std::endl;
    }
};

int main() {
    std::vector<int> values{1, 2, 3};
    std::unique_ptr<Printer> printer(new ConsolePrinter());

    for (const auto& value : values) {
        printer->print(value);
    }

    return 0;
}
```

这段代码里已经同时出现了很多关键主题：

- `vector` 表示动态数组
- `unique_ptr` 表示独占所有权
- `virtual` / `override` 表示多态接口
- `const auto&` 表示只读遍历且避免复制

如果这些点是分散记忆的，这段代码会看起来像“每一行都懂一点，但整体还是不敢改”。本文档的目标就是把这些点连成体系。

# 学习时最容易犯的错误

初学者最常见的误区通常不是太笨，而是顺序错了：

- 一上来就沉迷模板和高级技巧，基础模型没立住。
- 只记语法，不练编译和运行。
- 看见 `new/delete` 就以为这是默认写法。
- 把 `C++` 只当成“语法更复杂的 Java/Go/TS”。
- 把所有崩溃都归因于编译器或工具链，而不是先检查生命周期和边界。

如果你尽量按照“基础模型 -> 资源管理 -> 标准库 -> 面向对象”的顺序来学，会顺畅很多。

# 如何使用这套文档

最推荐的方式不是一口气从头读到尾，而是交替进行：

1. 阅读一章，先理解概念和推荐写法。
2. 手敲示例代码并编译运行。
3. 故意制造几个错误，观察编译器报什么。
4. 回头在项目里找类似代码。
5. 记录“这章内容在真实代码里通常长什么样”。

你会发现 `C++` 很多概念一旦和编译器、运行结果、真实工程代码绑在一起，理解速度会大幅提升。

# 本文档的边界

为了控制节奏，这套文档会刻意避免几件事：

- 不会把标准条款逐条展开。
- 不会把每个角落特性都讲成完整专题。
- 不会把所有旧风格代码一刀切说成“绝对错误”。
- 不会脱离项目场景去堆砌抽象定义。

它更像是一套“进入 `C++11` 工程的上手教程”，而不是一本标准参考书。

# 小结

你现在要做的不是“把一门完全陌生的语言从零背完”，而是把你已经有的开发经验迁移到 `C++11` 的世界里。只要先抓住构建模型、值与引用、对象生命周期、智能指针、标准库容器与算法、现代类设计这几条主线，进入项目代码会比想象中快很多。
