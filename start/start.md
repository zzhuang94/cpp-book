# 前言

如果你 10 年前系统学过 `C++98`，说明你其实已经拥有非常宝贵的基础：

- 你知道编译型语言是怎么回事。
- 你理解指针、引用、对象、构造/析构这些核心概念。
- 你接触过模板、`STL`、多态和头文件组织。

真正的困难通常不是“从零学 `C++`”，而是：

> 你脑子里保存的是一套老 `C++` 的工作记忆，但你接下来要面对的是一套已经明显现代化的 `C++11` 项目代码风格。

这套手册正是为这种场景准备的。

# 适用读者

本文档主要面向下面这类读者：

- 有编程经验，最好写过 `C/C++`
- 曾系统学习过 `C++98/03`
- 很久没有接触 `C++`
- 现在需要快速重新进入 `C++11` 项目开发

如果你完全没有编程基础，这套文档仍可阅读，但它默认你对“变量、函数、类、指针、引用、编译”这些概念至少不陌生。

# 文档重点

这不是一本追求“标准全覆盖”的手册，而是一套强调迁移效率的学习资料。重点不是把所有特性平铺罗列，而是帮你解决下面这些更现实的问题：

- `C++98` 里熟悉的写法，哪些在 `C++11` 里已经有更好的替代？
- 为什么现在项目里大量使用 `auto`、`lambda`、`nullptr`、智能指针？
- 如何从“手动管理资源”的思维，迁移到 RAII 和所有权表达？
- 怎样用更现代的方式理解类、继承、多态和接口？
- 接手真实项目时，哪些知识点最值得优先补齐？

# 写法风格

本文档会尽量统一采用下面这种风格：

- 中文讲解为主，英文关键字、库名、类型名保留
- 尽量给出短小、完整、可直接运行的示例代码
- 经常对比“旧写法”和“推荐写法”
- 重点解释“为什么现在更推荐这样写”
- 不把语法点孤立讲解，而是尽量贴近真实工程语境

# C++11 重点

`C++11` 是现代 `C++` 的分水岭。很多今天看起来理所当然的写法，几乎都从这一版开始成型：

- `auto`
- `decltype`
- 统一初始化
- 范围 `for`
- `nullptr`
- `enum class`
- `lambda`
- 右值引用
- 移动语义
- 智能指针
- `constexpr`
- `override` / `final`
- `=default` / `=delete`
- `chrono`
- 线程库
- `unordered_map`

如果你能把 `C++11` 真正学扎实，再往后看 `C++14/17/20` 会轻松很多；但如果 `C++11` 没有补齐，现代 `C++` 项目会让你产生很强的断层感。

# 阅读顺序

虽然侧边栏是按主题展开的，但如果你的目标是“尽快接手项目”，更推荐按下面顺序看：

1. 先看 `基础教程` 里的 `简介`、`环境准备`、`基础语法`、`变量`、`函数`、`指针`、`引用`。
2. 然后重点看 `常量`、`数组`、`字符串`、`数据结构`，把标准库的基本用法找回来。
3. 再进入 `面向对象`，重点关注 `类/对象`、`继承`、`多态`、`接口`。
4. 最后回头对照你的项目代码，把看到的特性一一映射到文档知识点上。

# 内容边界

为了提高学习效率，本文档会有意识避免以下倾向：

- 不追求把标准的每个边角都写全
- 不堆砌过多纯理论定义
- 不把历史遗留写法当作推荐实践
- 不把“能工作”和“推荐这样写”混为一谈

也就是说，你仍然会看到老写法，但通常是为了帮助你理解迁移，而不是鼓励你继续沿用。

# 断层示例

假设你记忆中的代码还是这样：

```cpp
#include <iostream>
#include <vector>

int main() {
    std::vector<int> values;
    values.push_back(1);
    values.push_back(2);
    values.push_back(3);

    for (std::vector<int>::iterator it = values.begin(); it != values.end(); ++it) {
        std::cout << *it << std::endl;
    }

    return 0;
}
```

这段代码没有错，但在 `C++11` 项目里，更常见的写法会是：

```cpp
#include <iostream>
#include <vector>

int main() {
    std::vector<int> values{1, 2, 3};

    for (auto value : values) {
        std::cout << value << std::endl;
    }

    return 0;
}
```

这里涉及的变化并不只是“代码变短了”，而是整套语言开始鼓励：

- 更直接的初始化方式
- 更清晰的遍历语法
- 更少样板类型书写

这就是这套手册想帮你建立的新直觉。

# 使用方式

最好的方式不是从头读到尾，而是交替进行：

1. 阅读某一章。
2. 把示例代码手敲一遍并编译运行。
3. 回头看你项目里的类似代码。
4. 记录“旧理解”和“新理解”的差异。

如果你能把“看文档”和“读项目代码”交替起来，这套手册的吸收速度会比纯阅读高得多。

# 小结

你现在面对的不是“重新学一门完全陌生的语言”，而是“把过去的 `C++98` 经验升级到 `C++11` 的工程现实里”。只要抓住 `auto`、`lambda`、移动语义、智能指针、现代类设计这些关键支点，进入项目的速度会远比你想象中快。
