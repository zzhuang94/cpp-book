# C++11 地图

很多人学 `C++11` 时容易陷入一种状态：

- 每个特性都见过一点
- 单独看大概能懂
- 放进真实项目里又很难形成整体判断

这篇文章的目标，就是先把 `C++11` 最关键的一张地图立起来。

它不是逐条解释所有新特性，而是回答两个问题：

1. `C++11` 到底带来了哪些真正改变工程写法的能力？
2. 如果你的目标是参与 **多人协作或维护期较长** 的 `C++` 代码库，哪些必须先掌握，哪些先知道存在就够？

----

# 时代位置

`C++11` 之前的很多代码风格，今天仍然能在老项目里看到；  
但从 `C++11` 开始，现代 `C++` 的主线才真正稳定下来。

最重要的变化不是“关键字变多了”，而是语言开始更明确地支持下面这些方向：

- 更清晰的对象生命周期表达
- 更自然的资源管理方式
- 更高效的大对象传递和返回
- 更现代的标准库和并发能力
- 更容易让编译器帮你检查设计意图

如果你把 `C++11` 学扎实，后面再看到 `C++14/17/20`，通常会觉得是在原有地图上继续扩展；  
如果 `C++11` 这一层没站稳，后面的新特性就很容易变成零碎记忆。

----

# 核心特性

## 1. `auto`

它的价值不是“少写字”，而是让代码把注意力放在变量语义上，而不是冗长类型名上。

```cpp
std::vector<std::string> names{"a", "b", "c"};
auto it = names.begin();
```

对工程代码来说，`auto` 最常见的价值是：

- 处理复杂迭代器类型
- 处理 lambda 对象
- 处理模板推导结果

## 2. `nullptr`

它解决的是“空指针常量语义不清”的问题。

```cpp
int* p = nullptr;
```

比起旧写法 `NULL`，`nullptr` 在重载解析和类型安全上更稳定。

## 3. 范围 for

它让容器遍历更接近真实意图。

```cpp
for (const auto& value : values) {
    std::cout << value << '\n';
}
```

它不是语法糖那么简单，因为它会推动你更多地使用标准容器和标准库风格。

## 4. 统一初始化

```cpp
int x{10};
std::vector<int> nums{1, 2, 3};
```

它帮助统一对象初始化风格，也能避免部分窄化转换。

## 5. `enum class`

它解决的是传统 `enum` 太容易污染作用域、太容易隐式转换的问题。

```cpp
enum class Color {
    Red,
    Green,
    Blue
};
```

## 6. lambda

很多现代 `C++11` 代码真正开始“像现代 C++”，就是从 lambda 普及开始的。

```cpp
auto is_even = [](int x) {
    return x % 2 == 0;
};
```

它会和算法、线程、回调、排序规则一起高频出现。

## 7. 移动语义

这是 `C++11` 最大的理解门槛之一，也是工程收益最大的能力之一。

你至少要建立两个直觉：

- 拷贝是复制一份资源
- 移动是转移资源所有权或资源内容

相关高频符号包括：

- `T&&`
- `std::move`
- 移动构造
- 移动赋值

## 8. 智能指针

`unique_ptr`、`shared_ptr`、`weak_ptr` 不是附属工具，而是今天读项目代码时判断所有权最重要的一组线索。

```cpp
std::unique_ptr<User> user(new User());
```

如果你不理解智能指针，就很难稳定地读懂现代 `C++11` 项目里的资源关系。

## 9. override/final

它们的价值是把“设计意图”直接写给编译器。

```cpp
class Base {
public:
    virtual void run() const = 0;
};

class Derived : public Base {
public:
    void run() const override {
    }
};
```

## 10. default/delete

这两者在现代类设计里非常重要。

- `=default`：明确保留默认行为
- `=delete`：明确禁止某个操作

它们会直接影响类的可构造性、可复制性和可移动性。

## 11. 线程库

`C++11` 把多线程正式纳入标准库：

- `std::thread`
- `std::mutex`
- `std::condition_variable`
- `std::atomic`
- `std::future`

即使你平时不写高并发服务，参与 **有一定体量的** 代码库时也很可能碰到这些能力。

## 12. `unordered_map`

它不是“新容器里的一种”，而是非常常见的日常主力。

对于很多“按 key 快速查找”的场景，它是你在工程里高频会看到的默认选择之一。

# 学习重点

如果你的目标是“接手项目”，优先级大致可以这样排：

## 第一层

- `auto`
- `nullptr`
- 范围 `for`
- lambda
- 右值引用与移动语义
- `std::move`
- 智能指针
- `override`
- `=delete` / `=default`

这些内容决定了你能不能顺利读现代 `C++11` 代码。

## 第二层

- `enum class`
- `constexpr`
- `noexcept`
- `using`
- 委托构造
- 成员默认值

这些内容会持续出现，但通常不是最早卡住你的点。

## 第三层

- 转发引用
- `std::forward`
- 更细的模板推导规则
- 原子内存序细节

这些在体量较大的代码库里也可能出现，但不必一开始就钻太深。

----

# 常见组合

你在项目里很少会单独看到某个特性，更多是看到它们组合出现。

例如：

```cpp
std::vector<std::unique_ptr<Handler>> handlers;

for (const auto& handler : handlers) {
    handler->run();
}
```

这段代码里已经同时包含了：

- `vector`
- `unique_ptr`
- 范围 `for`
- `const auto&`
- 多态调用

再比如：

```cpp
class Task {
public:
    Task() = default;
    Task(const Task&) = delete;
    Task(Task&&) noexcept = default;
};
```

这段代码里又同时涉及：

- `=default`
- `=delete`
- 移动语义
- `noexcept`

所以学习 `C++11` 的关键不是把每个词分散记住，而是逐渐熟悉这些组合在工程里通常意味着什么。

# 学习分组

你可以把 `C++11` 新特性分成三组来学：

## 语法升级

- `auto`
- `nullptr`
- 范围 `for`
- `using`
- `enum class`
- lambda

## 资源模型

- 右值引用
- `std::move`
- 智能指针
- `override`
- `=default`
- `=delete`
- `noexcept`

## 工程能力

- 线程库
- 标准容器扩展
- 构建系统中的 `-std=c++11`
- 调试、警告和诊断工具

这样学习时不容易把语言点和工程目标割裂开。

----

# 小结

`C++11` 最重要的不是“新增了多少语法”，而是它让现代 `C++` 的主线真正稳定下来：

- 用标准库而不是手工重复造轮子
- 用对象生命周期管理资源
- 用移动语义减少不必要复制
- 用类型系统和编译器表达设计意图

如果你先把这张地图建立起来，后面再进入指针、引用、智能指针、类设计、构建系统和项目阅读时，会顺畅很多。
