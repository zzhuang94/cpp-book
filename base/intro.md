# 简介

如果你 10 年前系统学过 `C++98`，再回来看 `C++11`，最大的感受通常不是“语法变多了”，而是语言设计的重心变了。`C++11` 不再只是给你更多技巧，而是在推动你用更安全、更明确、更现代的方式写代码。

本手册的目标不是把标准条文重新抄一遍，而是帮助你快速建立一套能直接上手 `C++11` 项目的知识框架：

- 重新拾起 `C++` 的基础语法和核心概念。
- 理解 `C++98/03` 到 `C++11` 的关键变化。
- 学会在工程里优先采用现代写法，而不是停留在旧习惯上。
- 通过可运行示例，把知识点尽快转成手感。

# 学习重点

接手 `C++11` 项目时，最值得优先补课的通常不是 `if`、`for` 这种基础语法，而是下面这些会直接影响工程代码可读性和正确性的能力：

- 类型推导：`auto`、`decltype`
- 初始化语法：统一初始化、成员默认值
- 资源管理：`unique_ptr`、`shared_ptr`、RAII
- 语义升级：右值引用、移动语义
- 可调用对象：`lambda`、`std::function`
- 面向对象增强：`override`、`final`、`=default`、`=delete`
- 标准库补强：`array`、`chrono`、线程库、无序容器

# C++98 差异

过去很多 `C++98` 代码都在和这些问题搏斗：

- 手动管理内存，容易泄漏。
- 接口表达能力弱，错误调用不容易在编译期暴露。
- 模板和类型书写冗长，影响开发效率。
- 临时对象复制成本高，性能优化依赖经验。

`C++11` 针对这些痛点提供了系统性改进：

- 用 `nullptr` 取代 `NULL`
- 用 `auto` 降低模板和迭代器书写成本
- 用范围 `for` 提升容器遍历可读性
- 用 `lambda` 替代大量临时函数对象
- 用移动语义减少不必要复制
- 用智能指针让资源所有权更明确
- 用 `override` 和 `=delete` 让接口意图更清晰

# 学习顺序

建议你按下面顺序阅读本手册：

1. 先补基础语法、类型、变量、函数、引用、指针。
2. 再重点攻克 `auto`、`lambda`、右值引用、移动语义、智能指针。
3. 然后回到类、继承、多态，重建“现代 C++ 面向对象”的理解。
4. 最后结合 `STL` 容器、算法和 I/O，把知识点串成真实代码风格。

# 最小示例

下面这段代码已经能体现几个 `C++11` 常用特性：列表初始化、`auto`、范围 `for`。

```cpp
#include <iostream>
#include <vector>

int main() {
    std::vector<int> values{1, 2, 3, 4, 5};
    int sum = 0;

    for (auto value : values) {
        sum += value;
    }

    std::cout << "sum = " << sum << std::endl;
    return 0;
}
```

编译运行：

```bash
g++ -std=c++11 intro_demo.cpp -o intro_demo
./intro_demo
```

# 学习原则

> 不要把 `C++11` 理解成“在 C++98 上新增了若干语法糖”。更合适的理解是：语言开始鼓励你把所有权、生命周期、接口意图、性能语义都写进代码里。

这也是为什么本手册会反复对比“旧写法”和“推荐写法”。你的目标不是记住所有新关键字，而是形成新的编码直觉。

# 小结

这套文档会以 `C++11` 为主线，但不会脱离 `C++98/03` 的背景。你会看到很多“以前怎么写，现在为什么不建议这样写，以及现在应该怎么写”的对照，这正是从老经验迁移到现代 `C++` 最高效的方式。
