# C++11 入门与工程上手手册

这套文档面向已经有其它语言开发经验的程序员，例如写过 `Go`、`PHP`、`TypeScript`，但几乎没有真实 `C/C++` 项目经验，或者学过很久以前的 `C/C++` 基础、现在已经记不清了。

它不是一本追求标准条文全覆盖的参考手册，而是一套强调“先建立正确心智模型，再进入项目代码”的教程。主线以 `C++11` 为主，同时会补充一些阅读老式 `C` / `C++98` 代码必须知道的基础背景。

# 这套文档适合谁

如果你符合下面任意一种情况，这套文档基本就是为你准备的：

- 你会写程序，但平时主要使用 `Go`、`PHP`、`JS`、`TS` 等语言。
- 你知道变量、函数、类、数组、指针这些词，但很多概念已经模糊。
- 你准备接手一个 `C++11` 项目，想先补一套能落地的基础。
- 你能看懂一点旧 `C++` 代码，但很难判断哪些写法已经过时。
- 你希望先学“工程里最常见、最值得优先掌握的内容”，而不是一上来啃标准。

如果你是完全零编程基础读者，也可以阅读，但这套文档默认你已经理解最基本的程序设计概念，例如分支、循环、函数、对象和模块化。

# 本文档想解决的问题

对很多其它语言开发者来说，进入 `C++` 最难的地方往往不是单个语法点，而是下面这些整体差异：

- 为什么同样是编译型语言，`C++` 里还要关心头文件、声明、定义和链接。
- 为什么 `C++` 里“对象离开作用域自动析构”会影响资源管理设计。
- 为什么指针、引用、值语义、移动语义会直接决定接口写法。
- 为什么现代 `C++` 不再鼓励大量手写 `new/delete`。
- 为什么标准库容器、算法、迭代器几乎构成了日常编码的主战场。
- 为什么很多旧教程里的写法虽然还能编译，但并不是今天推荐的工程实践。

这套文档会把这些问题拆开讲，并尽量用短小、完整、可运行的代码示例把抽象概念落到具体代码上。

# 文档主线

整套内容围绕三条主线展开：

1. 先重建基础语法和工程上下文。
2. 再建立现代 `C++11` 的资源管理和标准库使用习惯。
3. 最后进入类设计、继承、多态、接口这些面向对象主题。

其中最核心的不是背下所有关键字，而是建立下面这些直觉：

- 能用对象生命周期解决资源问题，就不要靠手工清理。
- 能把“谁拥有资源、谁只是借用”写清楚，就能减少大量 bug。
- 能用标准库表达意图，就不要急着自己造轮子。
- 能让编译器帮你发现错误，就不要把约束藏在注释里。

# 学习重点

如果你的目标是尽快接手 `C++11` 项目，建议优先掌握下面这些内容：

- 构建基础：源文件、头文件、编译、链接、`Makefile`、`CMake`
- 语言基础：类型、变量、常量、运算符、分支、循环、函数
- 资源模型：指针、引用、RAII、智能指针、移动语义
- 标准库：`string`、`array`、`vector`、`map`、`unordered_map`、算法
- 面向对象：类、继承、多态、接口、`override`、`=delete`
- 工程安全：异常、未定义行为、边界检查、调试与诊断

这些内容组合起来，基本覆盖了阅读和修改常见 `C++11` 业务代码的第一道门槛。

# 这不是一本什么样的文档

为了保证学习效率，这套文档会刻意避免几个方向：

- 不追求把所有标准细节都平铺展开。
- 不把历史写法和推荐写法混在一起讲。
- 不假设你已经有多年 `C++` 项目经验。
- 不把“能跑”误当成“适合长期维护”。

所以你会看到很多“旧写法可以工作，但现代项目更推荐另一种方式”的对比。这种对比不是为了否定历史，而是为了帮你尽快形成今天更常见的工程判断。

# 推荐阅读顺序

如果你是第一次认真进入 `C++11`，推荐按下面顺序阅读：

1. 先看前言和简介，建立整体学习地图。
2. 然后看环境、构建模型、基础语法、类型、变量、常量、函数。
3. 接着重点看字符串、数组、指针、引用、I/O、数据结构。
4. 再补异常、模板、并发、安全边界这些真实项目高频知识。
5. 最后进入类、继承、多态、重载、接口，建立现代 `C++` 面向对象心智模型。

如果你已经开始接触项目代码，也可以采用“文档一章 + 项目一段代码交替阅读”的方式，这往往比从头到尾纯读更高效。

# 如何使用这套文档

建议你用下面的节奏学习，而不是只看不练：

1. 阅读一章，先理解概念和推荐写法。
2. 把示例代码手敲一遍并编译运行。
3. 故意改几个地方，观察编译错误和运行结果。
4. 回头看真实项目里有没有类似代码。
5. 记录“这个写法在项目里意味着什么”。

`C++` 很多知识如果只停留在阅读层面，很容易觉得“好像懂了”；只有真正编译、报错、修改、再运行，直觉才会稳定下来。

# 阅读旧代码时的态度

你在真实项目里很可能同时遇到三类代码：

- 很老的 `C` 风格或 `C++98` 风格写法
- 已经部分现代化的 `C++11` 写法
- 更高标准下写出的 `C++14/17/20` 风格代码

这套文档会以 `C++11` 为中心，但也会补充你理解旧代码所需的背景。目标不是让你只会写最现代的新代码，而是让你能判断：

- 这段老代码在做什么
- 它有没有潜在风险
- 如果要逐步演进，第一步应该怎么改

# 最终目标

学完这套内容之后，你不一定已经成为资深 `C++` 工程师，但应该具备下面这些能力：

- 能独立搭建并编译最小 `C++11` 程序
- 能理解值、引用、指针、对象生命周期这些核心概念
- 能读懂常见标准库容器和算法的基本写法
- 能看懂现代 `C++11` 类、继承、多态接口的基本结构
- 能识别明显不安全或过时的写法
- 能在项目中做小到中等规模的代码修改，而不是完全无从下手

如果你把这套文档和真实项目代码结合起来使用，进入 `C++11` 工程的速度会比单纯看旧教程快得多。
