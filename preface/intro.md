# 简介

这套教程的核心目标，不是把 `C++` 讲成一份语法清单，而是帮助你建立一套能直接进入 `C++11` 项目的知识框架。

如果你来自 `Go`、`PHP`、`TS` 等语言，第一次认真学 `C++`，最常见的困惑往往不是“有没有循环和函数”，而是：

- 为什么这门语言这么强调对象生命周期。
- 为什么一个函数参数是值、引用还是指针，会影响接口设计。
- 为什么头文件、声明、定义和链接是日常工作的一部分。
- 为什么现代项目里很多人强调 `RAII`、智能指针、移动语义和标准库容器。

这一章的任务，就是先给你一张总地图，让后面的章节不至于变成零散知识点堆积。

# 不是 “语法更复杂的语言”

很多人会下意识地把 `C++` 套进自己熟悉的语言模型：

- 来自 `Go` 的读者，容易把它想成“更手动、更底层的编译型语言”。
- 来自 `PHP` 的读者，容易把它想成“类型更强、构建更重的后端语言”。
- 来自 `TS` 的读者，容易把它想成“语法复杂、但最终也是对象和函数组合”。

这些类比可以帮助入门，但也容易误导，因为 `C++` 有一个非常突出的特点：

> 它要求你显式思考 “值、对象、资源、生命周期和边界”，而这些概念在很多其它语言里经常由运行时系统帮你兜底。

所以学习 `C++` 最重要的第一步，不是记住多少关键字，而是接受这门语言的思维重点和你熟悉的语言并不完全一样。

# 要建立 5 条主线

整个 `C++` 基础阶段，可以浓缩成 5 条主线：

1. 程序是如何从源文件变成可执行文件的。
2. 类型、变量、表达式和控制流如何构成最基本代码。
3. 值、引用、指针和对象生命周期如何影响数据传递与资源管理。
4. 标准库容器、字符串、算法、I/O 如何组成日常业务代码。
5. 类、继承、多态和接口如何在现代风格下表达意图。

如果你能沿着这 5 条主线去阅读，后面的章节会更有方向感。

# C++11 的核心变化

相较于很多旧教程里常见的 `C++98/03` 风格，`C++11` 最重要的变化不是“又多了几条语法”，而是语言开始更系统地鼓励下面这些实践：

- 用 `nullptr` 替代 `NULL`
- 用 `auto` 降低冗长类型书写成本
- 用范围 `for` 提升容器遍历可读性
- 用 `lambda` 和算法库表达局部逻辑
- 用右值引用和移动语义降低不必要复制
- 用智能指针表达资源所有权
- 用 `override`、`=delete`、`=default` 明确接口边界
- 用 `chrono`、线程库、无序容器扩展标准库能力

> 这些变化的共同方向都是：让代码更明确、更安全、更容易让编译器帮你发现错误。

# 最值得优先补的知识

如果你现在的目标不是考试，而是“尽快能看懂并改动现有项目”，最值得优先补的通常不是最基础的 `if` 和 `for`，而是以下几类知识：

- 构建模型：编译、链接、头文件、声明、定义
- 所有权模型：谁创建资源、谁释放资源、谁只是借用
- 标准库基础：`string`、`vector`、`map`、`unordered_map`、算法
- 参数设计：值传递、`const T&`、`T&`、`T*`、`T&&`
- 类设计：初始化列表、默认函数、删除函数、RAII
- 常见风险：悬空引用、未初始化、越界、异常路径资源泄漏

这些内容会在真实项目里反复出现，掌握它们的收益远高于孤立背诵一些少见语法。

# 最小现代风格的示例

下面这段代码已经能体现几条很重要的现代 `C++11` 主线：列表初始化、范围 `for`、`auto`、标准库容器。

```cpp
#include <iostream>
#include <vector>

int main() {
    std::vector<int> values{1, 2, 3, 4, 5};
    int sum{0};

    for (const auto& value : values) {
        sum += value;
    }

    std::cout << "sum = " << sum << std::endl;
    return 0;
}
```

编译运行：

```bash
g++ -std=c++11 -Wall -Wextra -pedantic intro_demo.cpp -o intro_demo
./intro_demo
```

这段代码不复杂，但已经比很多旧风格写法多表达了几件事：

- 容器用花括号直接初始化。
- 遍历时优先使用范围 `for`。
- 遍历元素时用 `const auto&` 表示只读访问。
- 依赖标准库容器而不是手写动态数组。

# 几个关键词

下面这些词会在整套文档中不断出现，建议你先形成一个“够用的第一印象”：

## RAII

`RAII` 可以先粗略理解为：

> 让资源的获取和释放绑定到对象生命周期，而不是分散在业务代码里手工清理。

比如文件、锁、堆内存、网络连接、数据库句柄等资源，都可以借助对象构造/析构来管理。

## 所有权

所有权是现代 `C++` 理解代码的关键。你读到一个指针、引用或者对象时，要本能地问：

- 谁负责创建它
- 谁负责释放它
- 它是否可以为空
- 它是否只是临时借用

这会直接影响你该不该复制、移动、保存或删除它。

## 值语义与引用语义

- 值语义意味着复制一个对象时，得到的是独立副本；
- 引用/指针语义意味着多个地方可能共享同一个实体。

很多 API 设计差异都来自这两者的选择。

## 移动语义

当一个对象的资源可以被安全“转移”而不是“复制”时，`C++11` 可以借助右值引用和移动语义显著降低开销。这个主题在字符串、容器和类设计里都很重要。

## 标准库优先

现代 `C++` 的日常开发通常不是“从头手写链表、数组、字符串工具”，而是优先使用：

- `std::string`
- `std::vector`
- `std::array`
- `std::map`
- `std::unordered_map`
- `std::algorithm`
- `std::chrono`
- 智能指针

会熟练使用这套标准库组合，往往比会更多冷门语法更实用。

# 和旧风格的差别

旧教程里很常见下面这种代码：

```cpp
#include <iostream>
#include <vector>

int main() {
    std::vector<int> values;
    values.push_back(1);
    values.push_back(2);
    values.push_back(3);

    for (std::vector<int>::iterator it = values.begin(); it != values.end(); ++it) {
        std::cout << *it << std::endl;
    }

    return 0;
}
```

它当然可以工作，但在 `C++11` 项目里，通常更常见的是：

```cpp
#include <iostream>
#include <vector>

int main() {
    std::vector<int> values{1, 2, 3};

    for (const auto& value : values) {
        std::cout << value << std::endl;
    }

    return 0;
}
```

差别不只在“代码更短”，而在于现代写法更鼓励：

- 直接表达初始化结果
- 减少冗长类型样板
- 用更接近业务意图的遍历方式
- 降低手误和噪音

# 学习顺序

建议你按下面顺序阅读整套文档：

1. 先补 `环境准备` 和 `头文件与构建模型`，搞清楚程序怎么被编译出来。
2. 再看 `基础语法`、`基础类型`、`变量`、`常量`、`运算符`、`函数`。
3. 接着重点看 `字符串`、`数组`、`指针`、`引用`、`I/O`、`数据结构`。
4. 再补 `异常`、`模板`、`并发`、`安全边界` 等工程主题。
5. 最后进入 `类/对象`、`继承`、`多态`、`重载`、`接口`。

如果你想先快速进入项目，优先度最高的其实是：构建模型、函数、指针、引用、数据结构、类设计。

## 请优先做三件事

为了让这套文档真正产生效果，建议你养成下面这些习惯：

### 每章示例都自己编译

`C++` 是一门对编译错误、类型信息、构建细节非常敏感的语言。很多概念如果不亲自编译一次，很难真正形成手感。

### 每章都故意做一点破坏性实验

例如：

- 删除一个头文件，看编译器报什么。
- 把 `const auto&` 改成 `auto`，观察是否产生复制。
- 把 `nullptr` 改成 `NULL`，看看重载解析是否变化。
- 把值传递改成引用传递，理解行为差别。

这种实验比死记定义更有效。

### 反复问“这段代码的资源和生命周期是什么”

!> 这是其它语言开发者最容易忽略、但最值得提前养成的习惯。很多 `C++` bug 的根源，不是语法不会，而是生命周期判断错了。

# 常见误区

刚开始学 `C++11` 时，下面这些误区非常常见：

- 误以为智能指针只是“自动 delete 的语法糖”
- 误以为 `auto` 会让代码类型不安全
- 误以为所有大对象都必须传引用，忽略了值语义的价值
- 误以为只要程序能跑就说明生命周期没问题
- 误以为 `volatile` 能替代线程同步
- 误以为模板、异常、多态这些主题彼此完全独立

> 实际上，现代 `C++` 的很多设计都是互相联动的。比如你理解了 `RAII`，再看异常安全就顺很多；理解了值和移动语义，再看容器和函数接口也会更自然。

# 本章教会了你什么

读完这一章后，你至少应该记住：

- 学 `C++11` 的重点不是背语法，而是建立对象、资源和生命周期的直觉。
- 现代 `C++` 和很多旧教程写法相比，默认推荐风格已经发生明显变化。
- 标准库容器、算法、智能指针、`const`、`override` 等并不是附属技巧，而是主线。
- 后面的章节虽然分开讲，但它们会共同指向“如何在真实项目里写出清晰、安全、可维护的代码”。

- 这套教程会以 `C++11` 为主线，但不会脱离旧式 `C/C++` 背景。
- 你会看到很多“旧写法是什么、今天为什么不推荐、现代项目通常怎么写”的对照。
- 对其它语言开发者来说，这种方式通常比直接啃抽象语法树更高效，因为它更接近真实项目里的学习路径。
