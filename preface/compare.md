# 语言对照

`C++` 最容易学乱的地方，不是知识点太多，而是很多读者会下意识把它套进自己熟悉的语言模型里。
这样做可以帮助入门，但如果类比过头，也会让很多关键概念理解跑偏。

# 与 C 的关系

很多人会把 `C++` 简化成“带类的 C”，这个说法只在历史上勉强成立，在今天的工程实践里已经远远不够。

你可以先抓住下面几个事实：

- `C++` 保留了大量 `C` 语法和底层能力。
- 很多旧项目会同时混用 `C` 风格和 `C++` 风格代码。
- 现代 `C++11` 更强调对象、类型系统、标准库、RAII 和泛型能力。

换句话说，学习 `C++` 时确实需要理解 `C` 背景，但目标不是退回到“全部按 C 的方式写”，而是知道哪些历史包袱仍然会出现在项目里，哪些已经有更合适的现代写法。

# C 和 C++ 最重要的差别

如果你已经接触过一点 `C`，最值得先建立的差别感是：

1. `C` 更偏向过程式组织，`C++` 更强调类型与对象建模。
2. `C` 常靠手工管理资源，`C++` 更强调把资源绑定到对象生命周期。
3. `C` 标准库相对精简，`C++` 标准库提供了字符串、容器、算法、线程等大量日常能力。
4. `C++` 里的构造、析构、拷贝、移动会直接影响接口设计和资源安全。

例如在 `C` 里，文件、锁、内存往往需要你手动成对处理；而在现代 `C++` 里，更推荐把这些资源交给对象析构自动回收。

# 与 Go 的差异

如果你来自 `Go`，最容易误判的通常不是语法，而是资源和并发模型：

- `Go` 有垃圾回收，`C++` 没有统一运行时替你管理所有资源。
- `Go` 中切片、map、goroutine 是非常强的默认工具；`C++` 里更多依赖容器、对象生命周期和显式同步。
- `Go` 常鼓励通过 channel 组织并发；`C++11` 并发更接近线程、锁、原子类型这些底层工具。

所以从 `Go` 迁移到 `C++` 时，一个很重要的调整是：

> 不要默认相信运行时会兜底，很多资源释放和并发正确性都要你自己写清楚。

# 与 PHP 的差异

如果你来自 `PHP`，最明显的变化通常出现在构建、类型和运行模型上：

- `PHP` 很多场景是解释执行或由运行环境统一加载。
- `C++` 需要显式编译、链接，并理解头文件和源文件的关系。
- `PHP` 里对象和数组的使用体验与 `C++` 标准库容器差别很大。
- `C++` 更强调值语义、拷贝成本、引用和对象生命周期。

对 `PHP` 开发者来说，最值得尽早建立的直觉是：`C++` 不是“类型更强的后端脚本语言”，它的很多设计都和构建过程、资源模型、性能边界直接相关。

# 与 JavaScript / TypeScript 的差异

如果你来自 `JS/TS`，最需要尽早接受的几件事是：

- `C++` 不是动态对象系统，类型在编译期就会深度影响代码组织。
- `TS` 的类型更多服务于开发期校验，`C++` 的类型同时深度参与编译、重载、模板和对象布局。
- `JS/TS` 里常见的异步模型不是共享内存多线程；`C++11` 并发则需要显式面对共享状态同步。
- `C++` 里空指针、引用有效性、对象生命周期错误，往往不是简单异常，而可能直接进入未定义行为。

因此，`C++` 不能简单理解为“语法更复杂的 TS”。它要求你对对象真实存在的方式、销毁时机和内存边界有更稳定的判断。

# 与 Java / C# 的差异

如果你有 `Java` 或 `C#` 背景，也要注意一个常见误区：不要把 `C++` 的类直接等同于托管语言对象。

区别主要在这里：

- `C++` 对象既可以在栈上，也可以在堆上。
- 局部对象离开作用域就会析构，这会直接影响资源释放设计。
- 参数传值、传引用、传指针不只是语法形式，而是语义选择。
- 多态只是 `C++` 的一部分，不是唯一主线。

很多刚从托管语言进入 `C++` 的读者，会把所有东西都想写成“堆对象 + 指针到处传”，这通常不是现代 `C++11` 的推荐路线。

# 最容易忽略的 6 个差异

无论你来自哪种语言，下面 6 点都值得优先记住：

1. 作用域不仅影响可见性，也影响析构时机。
2. 裸指针不默认表示拥有资源。
3. 值传递、引用传递、指针传递会改变接口语义。
4. *头文件、声明、定义、链接是日常基础，不是边角知识。*
5. 标准库容器和算法是主力，不是附属工具。
6. 程序能跑不代表写法安全，未定义行为可能长期潜伏。

# 应该怎样用类比

和其它语言做对照是有价值的，但更推荐这样使用类比：

- 用类比理解入口概念。
- 用实际 `C++` 代码修正类比误差。
- 一旦发现类比解释不了生命周期、所有权和构建问题，就以 `C++` 自己的模型为准。

类比的作用是帮助你起步，不是替代 `C++` 心智模型本身。

# 建议你优先建立的心智模型

如果你现在时间有限，先把下面这几条立住，后面会顺很多：

- `C++` 的核心难点不只是语法，而是资源、生命周期和构建模型。
- 现代 `C++11` 不是“继续写更复杂的 C”，而是有自己清晰主线的工程语言。
- 旧代码很常见，但旧写法不等于推荐写法。
- 学会标准库、RAII、值语义和接口边界，比死背冷门语法更重要。

# 小结

- 理解 `C++` 最有效的方式，不是强行把它塞进某个熟悉语言的框架里，而是在对照中逐步建立它自己的模型。
- 只要你开始把注意力放到对象生命周期、资源所有权、头文件与构建、标准库和接口设计这些主线上，后面的章节就会越来越成体系。
