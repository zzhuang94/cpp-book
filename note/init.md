# 几种初始化

在 C++ 里，下面三种写法都很常见：

```cpp
int i(2);
int ii{3};
int iii = 4;
```

> 对 `int` 来说，这三种写法有什么差别

它们都能把变量初始化出来，但语法来源、适用场景和一些细节并不完全一样。

> 想强调“按参数构造对象”，常用 `()`；  
> 想避免窄化转换，优先考虑 `{}`；  
> 简单基本类型赋值风格，`=` 也完全可以。

不过要特别注意：`{}` 并不是任何场景都无脑更好；像 `std::vector` 这种类型，`{}` 和 `()` 的语义可能不同。

---

# 直接初始化 ()

```cpp
int i(2);
std::string name("cpp");
```

这种写法叫**直接初始化**。它的特点是：
- 看起来像“把参数交给构造函数”
- 对类类型很常见
- 在成员初始化列表里也经常见到

```cpp
class Buffer {
public:
    Buffer(std::string name, std::size_t size)
        : name_(std::move(name)), size_(size), data_(size) {
    }

private:
    std::string name_;
    std::size_t size_ = 0;
    std::vector<int> data_;
};
```

这里的 `data_(size)` 不是“把一个 `size_t` 赋值给 `vector`”，  
> 而是“调用 `std::vector<int>` 的构造函数，用 `size` 创建一个指定大小的 `vector`”。

```cpp
std::vector<int> v(3);
// 表示创建 3 个元素的 `vector`，内容相当于：
// {0, 0, 0}
```

---

# 列表初始化 {}

```cpp
int ii{3};
std::string title{"hello"};
```

这种写法叫**列表初始化**，也常被叫做“统一初始化”。它最大的优点是：**禁止窄化转换**。

```cpp
int a = 3.14;  // 可以，结果是 3
int b(3.14);   // 可以，结果是 3
int c{3.14};   // 错误，编译器会阻止
```

这里 `3.14` 是 `double`，转成 `int` 会丢失小数部分。 **`{}` 会把这种风险拦下来，现代 C++ 代码更偏爱它。**

> 因此在日常开发里：
> - 如果你想让初始化更安全，优先考虑 `{}`
> - 尤其是在数字类型之间转换时，`{}` 很有价值

---

# 拷贝初始化 =

```cpp
int iii = 4;
std::string lang = "C++";
```

这种写法叫**拷贝初始化**。

对内置类型来说，它通常最直观，也最像“赋值”。
> 这里本质上仍然是 **初始化**，*不是“先创建再赋值”*。

```cpp
int x = 10;
```
> 直接把 `x` 初始化成 `10`。

对简单类型来说，`=` 很自然；但在类类型里，它可能触发不同的构造函数匹配规则，所以不能简单理解成普通赋值。

---

# vector 里 () 和 {}

这是最值得单独记住的一点。

```cpp
std::vector<int> a(3);   // {0, 0, 0}
std::vector<int> b{3};   // {3}
```

- `a(3)` 表示创建一个大小为 3 的 `vector`
- `b{3}` 表示创建一个只含 1 个元素的 `vector`，这个元素的值是 3

---

# 初始化列表里的写法

构造函数里常见这种写法：

```cpp
Buffer(std::string name, std::size_t size)
    : name_(std::move(name)), size_(size), data_(size) {
}
```

冒号后面这一段叫**成员初始化列表**。*它做的事情 `不是`：*

```cpp
name_ = std::move(name);
size_ = size;
data_ = std::vector<int>(size);
```

> 而是：
> - `name_` 在对象创建时直接构造出来
> - `size_` 在对象创建时直接构造出来
> - `data_` 在对象创建时直接构造出来

这通常比“先默认构造，再在函数体里赋值”更自然，也通常更高效。

---

# 小结

三种初始化方式都合法，但关注点不同：

- `()` 更像“传参数构造对象”
- `{}` 更强调安全，尤其能防止窄化转换
- `=` 对简单类型很自然，但在类类型里要注意它的初始化语义

现代 C++ 里，`{}` 的使用频率确实更高；  
但遇到 `std::vector`、`std::initializer_list` 相关类型时，一定要留意 `()` 和 `{}` 的含义是否一致。

若构造函数形参类型为 `std::initializer_list<T>`，则 **不得** 将形参所绑定的列表在对象内部以指针或引用形式长期保存：列表由实现所管理的临时存储支撑，构造结束后即失效，延续使用构成未定义行为。
